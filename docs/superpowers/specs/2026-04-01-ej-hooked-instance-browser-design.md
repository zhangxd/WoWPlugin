## 2. 目标（收窄后）

1. **集成形态（B）**：在 **暴雪冒险手册** 原有 **左侧实例列表 + 右侧预览/详情** 流程上挂接；**不**提供与手册等价的独立备用窗口；Hook 失败时 **`Toolbox.Chat` 一行提示**（经 `Toolbox.Chat`，符合 AGENTS）。
2. **坐骑相关筛选**（见 §4）：  
   - **实例列表**：在资料片相关控件旁增加 **「仅坐骑掉落」** 勾选，筛 **当前资料片副本列表**。  
   - **副本详情 · 战利品**：在战利品区域 **叠加**「仅坐骑」类筛选（**不**在暴雪战利品 **Tab 栏** 增加原生第四类标签），见 §4.1。
3. **CD 融合预览**（分两层，见 §5）：**首页** 为 **当前选中资料片内全部副本** 列表，**每个副本元素** 展示 **各难度** 的 CD 与进度；**详情页** 在标题旁展示 **当前难度** 的 CD 与进度；数据与 `GetSavedInstanceInfo` / `GetSavedInstanceEncounterInfo` 一致，按 **副本名** 或 **journalInstanceID** 映射匹配。
4. **工程约束**：`RegisterModule`、`ToolboxDB.modules.<moduleId>`；玩家可见字符串 **Locales.lua**；**禁止**在模块内直接 `DEFAULT_CHAT_FRAME`；坐骑判定经 **`Toolbox.MountJournal`** 或既有封装；**Core 不堆业务**。

## 4. 坐骑筛选（仅实现项）

- **语义**：勾选 **「仅坐骑掉落」** 后，左侧（或插件接管后的）实例列表 **只保留**「手册 loot 扫描中可解析出坐骑物品」的实例。
- **数据策略（混合 C，与 brainstorming 一致）**：
  - **默认**：使用 **持久化缓存**（`journalInstanceID` → 是否含坐骑），键内带 **版本/TOC 失效** 字段，避免旧缓存长期错误。
  - **无缓存或 Shift**：首次勾选或 **Shift + 点击** 勾选框触发 **全量重扫**（分帧/分批，避免单帧卡顿）；可 **Chat 短提示**「正在构建坐骑索引…」（文案键）。
  - **扫描**：经 `Toolbox.EJ` + 既有 `ScanInstanceDetails` 或等价 **仅判坐骑** 的轻量路径（实现时择优，避免重复全量宠物/玩具解析若不需要）。

### 4.1 副本详情 · 战利品标签区域（仅坐骑展示）

- **目标**：在玩家打开 **副本详情** 并切换到暴雪 **「战利品」** 子界面时，提供 **只关心坐骑掉落** 的筛选能力，与 §4 坐骑判定逻辑 **复用**（`C_MountJournal` / 物品→坐骑 ID 等经 `Toolbox.MountJournal` / 既有封装）。
- **不可行 / 不做**：向暴雪战利品面板 **原生 Tab 行** 注册 **与官方并列的新「坐骑」Tab**（客户端无公开 API；也不建议篡改私有 Tab 模板）。
- **可行 UI**：在战利品内容区 **上方或侧方** 用插件 **`CreateFrame` 叠加** 控件（如 **`CheckButton`「仅显示坐骑」**）。勾选后：  
  - **优先方案**：对当前实例已展示的 loot 行 **隐藏非坐骑**（需能枚举到 itemID 并与坐骑判定）；或  
  - **备选**：Hook 暴雪刷新战利品列表后的回调，再过滤显示（**易碎**，须 **`pcall`**、版本注释、失败则仅隐藏插件勾选不报错）。
- **与列表筛的关系**：列表筛是 **实例级**「是否有坐骑」；战利品筛是 **当前副本战利品网格内** 的 **物品级** 过滤，二者互补。
- **文案**：全部走 `Locales.lua`（如「仅显示坐骑」），与 §4 其它键区分或复用前缀。

## 5. CD 融合预览

### 5.1 数据匹配

- 遍历 `GetNumSavedInstances` / `GetSavedInstanceInfo`，与手册实例 **显示名** 或 **`journalInstanceID`**（经 `Toolbox.EJ` / 映射层）对齐。
- **多难度**：同一副本名在存档中可对应 **多条** `GetSavedInstanceInfo`（每条不同 **`difficultyId`**）。首页须在 **每个副本元素** 上 **逐难度** 展示（见 §5.2）；实现时用 **`difficultyId`** + **`difficultyName`**（API 返回）区分档位，与游戏内地下城/团队界面锁定 **一致**。
- **聚合键**：以 **副本实例**（手册行）为主键，将锁定行 **按副本匹配** 后 **再按难度** 分组；同副本多难度 **不得** 合并成单条 x/x。

### 5.2 冒险指南首页（实例列表层）

- **列表范围**：**当前在冒险手册中选中的资料片（Tier）** 下，`GetNumInstances` / `GetInstanceByIndex` 能枚举到的 **全部副本**，与暴雪左侧（或等价）列表 **一一对应**；切换资料片后列表随之变为 **新资料片下的全部副本**。
- **位置**：在上述 **每一个副本列表元素** 上（名称区域旁或名称下方），增加插件展示区。
- **每个难度的展示**：对 **该副本在锁定数据中出现的每一种难度**（每条 `GetSavedInstanceInfo` 一条），分别展示：  
  - **难度名称**（使用 API 提供的 `difficultyName` 或本地化等价）；  
  - **剩余刷新时间**；  
  - **首领进度** **`x/x`**（`encounterProgress` / `numEncounters`）。  
- **布局**：**不再** 限定为「英雄 | 史诗」两栏；采用 **逐难度多行**（或紧凑多行块），使 **每一种有记录的难度** 均有独立一行（或独立单元）。若某副本在存档中 **仅有** 英雄、史诗两条，则 **只显示两行**；若另有随机团、普通等锁定，则 **同样各占一行**，直至覆盖该角色对该副本 **全部** 难度锁定。
- **无某难度锁定**：该难度 **不出现** 单独一行，或显示 **「难度名 + —」**（实现计划二选一，Locales 统一无锁定短文案）。
- **无锁定副本**：该副本元素下可 **仅显示标题无副文本**，或一行 **「无锁定」**（Locales）。
- **边界**：手册存在但角色从未进本则无锁定行；**大秘境（钥石）** 等是否与团本/地下城锁定 **同表**，以正式服 `GetSavedInstanceInfo` 实际行为为准，实现时单独说明。

### 5.3 副本详情（进入某一实例后）

- **位置**：**副本标题/副本名字** **同一行尾部**（或紧跟名字之后），**内联** 展示 **CD** 与 **首领进度**（**x/x**）。
- **与难度的关系**：详情页内暴雪 **难度选择器** 决定当前上下文；此处展示 **当前所选难度** 下的 **剩余刷新时间** 与 **x/x**（从锁定列表中匹配 **当前难度** 的那一条）。切换难度时 **随选随更**（若 API 可侦听；否则在显示刷新时更新）。
- **「CD」语义**：与首页一致，指 **剩余刷新时间**（或等价表述如「重置：…」），避免与技能 CD 混淆，Locales 用统一前缀。

### 5.4 通用

- **层级**：`FrameStrata` / 锚点 **不遮挡** 暴雪关键按钮与可点击区；随版本用 **pcall**，失败则 **不显示** 插件 CD 文本而非报错。

## 6. 模块与持久化

- `moduleId`：可与现有 `saved_instances` 合并或重命名（实现计划定）；至少包含：  
  - `mountFilterEnabled`（bool）、  
  - 坐骑缓存表 + **版本戳**、  
  - 可选「在手册显示扩展 UI」开关。  
- **Mover**：若 **无独立窗口**，则 **不** 注册大块拖动；仅手册内子控件。

---

## 7. 入口

- **主路径**：打开游戏 **冒险手册** 即见扩展控件与 CD 预览。  
- **设置**：保留模块 **总开关**（关闭时 **不挂载** Hook、不建控件）；**不**要求 `/toolbox instances` 打开独立窗口（若保留斜杠，行为改为 **打开冒险手册** 或 **无操作**，实现计划定）。

---

## 8. 风险与验收

- **风险**：手册 Frame 名变更、右栏锚点偏移；坐骑缓存与补丁不同步。  
- **验收**：  
  - 勾选「仅坐骑」后 **实例列表** 与游戏内坐骑掉落认知一致（抽样）；**战利品** 勾选「仅显示坐骑」后网格与预期一致（或降级无报错）；  
  - Shift 重扫与缓存生效；  
  - 首页为 **当前资料片下全部副本**；**每个副本元素** 上 **各难度** 的 **重置时间** 与 **x/x** 与游戏内锁定一致；详情页 **当前难度** 下名字后的 CD 与 **x/x** 一致；  
  - Hook 失败仅 **Chat 一行**，无 Lua 刷屏。

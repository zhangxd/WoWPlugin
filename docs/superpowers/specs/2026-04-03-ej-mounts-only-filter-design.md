# 冒险指南「仅坐骑」副本列表筛选 — 设计规格

**状态**：已定稿；当前实现改为依赖 `Toolbox.DungeonRaidDirectory` 的**登录预热缓存**，对已知假阴性副本做**名称补源**，并在勾选时使用**自绘覆盖列表**显示筛选结果；目录构建会按**当前冒险手册资料片优先**调度，玩家浏览冒险手册地下城 / 团队副本列表时，目录后台扫描仍会暂停，待离开该语境后自动恢复。
**日期**：2026-04-03
**范围**：仅「地下城 / 团队副本」语境下、资料片下拉左侧复选框 + 列表按坐骑掉落筛选 + 开关聊天提示；**不**含副本详情区 CD 行（另案）。

---

## 1. 目标与验收

| 项 | 说明 |
|----|------|
| **入口** | 打开冒险指南，进入会显示 **资料片下拉 + 副本列表** 的页面（地下城 / 团队副本相关页签；以实机 `instanceSelect` 带 `ExpansionDropdown` 为准）。 |
| **控件** | 在 **资料片选择控件左侧** 增加一个 **「仅坐骑」** 复选框（不移动、不替换暴雪原有控件）。 |
| **聊天** | 勾选/取消勾选时，各输出 **一行** 经 `Toolbox.Chat` 的提示（文案走 `Locales`）。 |
| **筛选** | 仅当**当前页签 + 当前资料片**下的**整份副本列表**都已得到坐骑掉落结论时，复选框才可点击；勾选后，列表区改由**自绘覆盖列表**接管，只显示已判定 **有坐骑掉落** 的副本，并按当前资料片列表顺序重新紧凑排列。 |
| **刷新中** | 若当前列表仍存在 `HasAnyMountLoot(jid) == nil` 的副本，则复选框**置灰不可切换**，鼠标悬停 Tooltip 提示“正在刷新”或“为避免与当前浏览打架已暂停，关闭界面后继续”。 |
| **恢复反馈** | 若目录层构建曾因玩家打开该列表而暂停，则在离开列表语境并恢复完成后，聊天框额外提示一次「仅坐骑筛选已可用」。 |
| **构建优先级** | 目录层默认仍会后台补齐全部资料片；但当玩家打开某个资料片的地下城 / 团队副本列表时，会把该资料片的待处理副本调到当前构建队列前面。 |
| **边界** | **不**新增其它界面元素（无列表行角标、无详情区补丁）；**不**改暴雪默认布局除上述锚点外。 |

## 2. 架构与模块契约

| 项 | 约定 |
|----|------|
| **模块 id** | `ej_mount_filter`（稳定字符串，作 `ToolboxDB.modules.ej_mount_filter` 与 TOC 文件名对应）。 |
| **注册** | `RegisterModule`：`OnModuleLoad` 内注册 `ADDON_LOADED` → `Blizzard_EncounterJournal` 后初始化挂接；`PLAYER_LOGIN` 无需特殊逻辑时可空。 |
| **数据** | `ToolboxDB.modules.ej_mount_filter`：`enabled`、`debugChat`；副本掉落结论统一读取 `ToolboxDB.global.dungeonRaidDirectory` 的摘要缓存；目录层允许对已知假阴性副本做小型补源表兜底；模块自身不再维护会话扫描表。 |
| **门面** | 数据源：`Toolbox.DungeonRaidDirectory`；聊天：`Toolbox.Chat`；**禁止**模块内 `EJ_*` 直调、`DEFAULT_CHAT_FRAME`。 |

## 3. 实现路径对比（已选推荐）

| 方案 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **共享目录摘要 + 名称补源 + 自绘覆盖列表**（**已采用**） | `Toolbox.DungeonRaidDirectory` 登录后后台预热基础目录 + 支持难度 + `hasMountLoot` 摘要；对已知 EJ 假阴性副本做名称补源；当前冒险手册资料片优先扫描；勾选时由模块自绘覆盖列表显示过滤结果 | 统一数据源、可复用；能兜底已知假阴性；不再受原生列表空槽位、分页边界、池化行复用影响 | 需要一次预热构建；补源表需要少量人工维护 |
| 模块私有同步扫描 | 勾选时本模块直接切 EJ 并扫战利品 | 实现集中在一个文件 | 会卡帧、污染 EJ 共享选中状态、难以复用 |
| 静态全量 ID 表 | 插件维护完整「有坐骑」副本表 | 运行时零扫描 | 维护成本高、易过期，且难覆盖多资料片 / 赛季变化 |

**采用共享目录摘要 + 名称补源 + 自绘覆盖列表**：目录层在登录后异步预热，过滤模块仅消费 `HasAnyMountLoot(journalInstanceID)`；对已知假阴性副本允许目录层按归一化名称补源为 `true`；当玩家进入冒险手册的某个资料片列表时，该资料片会被提升为当前构建优先级；单个副本一旦在某个难度命中坐骑，则停止继续扫描该副本剩余难度。`nil` 表示“当前列表仍未就绪”，因此复选框保持灰显且列表不做过滤，直到该列表所需数据齐全。勾选后不再隐藏原生行，而是以覆盖列表重绘过滤结果。

## 4. UI 挂接与生命周期

1. **`Blizzard_EncounterJournal` `ADDON_LOADED`**（或已加载时同步初始化）：
   - 对 **`EncounterJournal.instanceSelect`** 各 **`HookScript("OnShow", …)`** 一次（去重）。
   - **`hooksecurefunc("EncounterJournal_ListInstances", …)`**（若全局存在）：回调内 **尝试创建复选框**、**尝试创建覆盖列表**、**刷新复选框可用态**、**刷新覆盖列表显示**，并同步目录层的暂停 / 恢复状态。
2. **创建时机**：仅在 `ExpansionDropdown` 已存在时 `CreateFrame` 复选框；父级为 `instanceSelect`。
3. **锚点**：
   - 复选框置于 **`ExpansionDropdown` 左侧**（例如 `CheckButton` 的 `RIGHT` 对齐 `ExpansionDropdown` 的 `LEFT`，带固定偏移）；标签文字在复选框左侧或小标题区，**不**挤压其它子控件——若版本布局冲突，以 `pcall` 锚点失败则静默跳过并可在调试模式记日志（正式服默认静默）。
   - 覆盖列表锚到原 `instanceSelect.ScrollBox`（或等价列表区）矩形范围，激活时接管该区域显示；原生列表本身不再负责“按行隐藏”。
4. **禁止**：以 **固定秒级 `C_Timer.After`** 作为「等布局」的**唯一**手段；允许 `C_Timer.After(0, …)` 仅作同帧合并且注释用途（见 AGENTS）。
5. **目录构建时机**：`Toolbox.DungeonRaidDirectory` 在登录后按需自动预热；`ej_mount_filter` **不**在玩家浏览副本列表时主动 `StartBuild(false)`。
6. **暂停 / 恢复**：当玩家进入地下城 / 团队副本列表语境且目录层仍在构建时，模块调用 `PauseForEncounterJournal()`；关闭冒险手册，或离开该列表语境后，再调用 `ResumeForEncounterJournal()`。
7. **优先级刷新**：模块在地下城 / 团队副本列表语境下，会持续调用目录层的“当前资料片优先”接口；目录层仅重排**未处理的尾部队列**，不会打断当前正在扫描的副本。

## 5. 目录摘要与筛选逻辑

1. **数据来源**：过滤模块只读取 `Toolbox.DungeonRaidDirectory.HasAnyMountLoot(journalInstanceID)`；目录层可对已知假阴性副本按名称补源。
2. **当前列表就绪判定**：模块收集**当前页签 + 当前资料片**的全部 `journalInstanceID`；只有它们全部满足 `HasAnyMountLoot(jid) ~= nil` 时，复选框才视为可用。
3. **覆盖列表数据源**：覆盖列表按 `Toolbox.EJ.GetInstanceByIndexFlat(index, isRaid)` 枚举**当前资料片整份列表**，保留原列表顺序，但**不复用原生分页结果**；因此筛选后会重新紧凑排列，避免第一页空、翻页后才出现的问题。
4. **可见性**：只有在“挂件可见 + 复选框已勾选 + 当前列表已就绪”三者同时满足时，才显示覆盖列表；覆盖列表仅渲染 `HasAnyMountLoot(jid) == true` 的副本，点击行后调用 `Toolbox.EJ.SelectInstance(jid)`，让原生详情区继续工作。
5. **未就绪时**：复选框灰显，Tooltip 提示刷新状态；即使存档里保持勾选偏好，也**不**立即接管列表显示。
6. **状态恢复**：EJ 选中状态的捕获 / 恢复由 `DungeonRaidDirectory` 内部私有 `EJStateDriver` 负责，本模块不再维护会话扫描状态，也不再主动切换难度或扫描战利品。
7. **单副本提前停止**：目录层在坐骑摘要阶段按支持难度顺序扫描；若某个副本已由补源判为 `true`，或某个难度命中坐骑，则该副本剩余难度直接视为“逻辑完成”，不再继续扫。

|------|------|

## 7. 设置（可选首版）

- **最低**：无设置页，仅冒险指南内复选框 + 存档 `enabled`；坐骑判定统一读取共享目录缓存。
- **若加 `RegisterSettings`**：总开关「启用冒险指南坐骑筛选」、归入 `settingsGroupId = "misc"`；打开设置时再读 `Locales` 说明。

**首版建议**：模块 **`RegisterSettings` 仅最小项**（例如总开关）或 **暂不注册设置页**，避免范围膨胀；以规格评审为准。

## 9. 风险与边界

- **手册 Frame 变更**：`instanceSelect` / `ExpansionDropdown` 改名会导致控件创建失败 → 静默降级，不影响手册使用。
- **目录补源维护**：名称补源只应用于已确认的假阴性副本，条目应保持小而明确，避免演化成静态大全表。
- **原生分页控件**：覆盖列表接管后，原生分页的视觉和交互语义可能不再等价；实现上应优先保证筛选结果正确、无空槽位、可滚动浏览。
- **首次构建体验**：若登录后后台预热尚未覆盖到当前列表，玩家打开该列表时复选框会暂时不可用；此时需关闭手册或离开该列表，让后台继续构建。
- **与总设计**：更广义的「CD 行」「独立面板」见 [2026-04-03-adventure-guide-mount-filter-lockout-redesign.md](../../specs/2026-04-03-adventure-guide-mount-filter-lockout-redesign.md)，与本模块独立。

# 冒险手册任务页签状态视图改造设计

- 日期：2026-04-11
- 状态：已确认，可执行
- 关联模块：`encounter_journal`
- 适用范围：`Toolbox`（Retail）

---

## 1. 背景

当前 `encounter_journal` 任务页签的 `状态` 视图沿用“左侧地图树 + 右侧三段泳道（可交付 / 进行中 / 待解锁）”结构。

本次改动只调整 `状态` 视图，使其更直接回答“当前该做什么”：

1. 左侧不再显示地图树。
2. 左侧改为“当前任务列表”。
3. 右侧改为展示当前所选任务所属的完整任务线。
4. 右侧任务线中高亮当前选中的任务节点。

`类型 / 地图` 视图保持现状，不在本次范围内调整。

---

## 2. 已确认决策

1. 模块归属：沿用现有模块 `encounter_journal`，不新增模块、入口或 TOC。
2. 改动对象：仅改 `状态` 视图。
3. 左侧内容：完全替换为“当前任务列表”，不再保留地图树过滤。
4. 当前任务口径：显示“当前可做”的任务，即：
   - `readyForTurnIn == true`
   - 或 `status == "active"`
   - 明确不显示 `pending`
5. 点击左侧任务后的右侧内容：
   - 显示该任务所属的完整任务线
   - 当前任务节点保持选中高亮
6. 右侧详情区行为：
   - 在 `状态` 视图下，选中任务时不进入纯文本详情页
   - 统一显示任务线任务列表
7. 边界：
   - 不改 `类型 / 地图` 视图
   - 不改任务排序规则以外的交互
   - 不改存档键结构
   - 不新增设置项

---

## 3. 设计方案

### 3.1 左侧状态视图列表

`状态` 视图左侧使用独立的数据构造逻辑，不复用地图树节点。

左侧列表项直接渲染任务节点，每项至少携带：

- `questID`
- `questLineID`
- `mapID`
- `selected`
- `text`

左侧数据源口径已确认调整为：

- 以**角色任务日志中的全部当前任务**为准
- 不再要求任务必须存在于 `InstanceQuestlines` / 任务线模型中
- 只要该任务在当前 Quest Log 中，就应出现在左侧列表

文案使用现有任务状态前缀：

- 可交付：沿用 `readyForTurnIn`
- 进行中：沿用 `status == "active"`

默认排序采用：

1. 可交付优先
2. 进行中其次
3. 同状态内按 Quest Log 原始顺序稳定输出

### 3.2 右侧状态视图内容

当 `selectedView == "status"` 且已选中有效 `questID` 时：

1. 通过 `GetQuestDetailByID(questID)` 解析所属 `questLineID`
2. 通过 `GetQuestListByQuestLineID(questLineID)` 取完整任务线
3. 右侧按现有任务行列表样式渲染整条任务线
4. 当前 `questID` 对应节点高亮

右侧标题优先显示任务线名；若任务线名不可用，则回退到现有任务列表标题文案。

若左侧任务存在于 Quest Log，但**没有任务线映射**：

1. 左侧仍然显示该任务
2. 右侧回退为纯任务详情文本
3. 不显示“未找到任务线”空态

### 3.3 选择状态

不新增存档键，继续复用现有：

- `questViewSelectedQuestLineID`
- `questViewSelectedQuestID`
- `questViewSelectedMapID`

其中：

- `状态` 视图进入时优先恢复上次选中的 `questID`
- 若上次任务已不再属于“当前任务”集合，则自动降级到当前列表第一项
- 若当前任务列表为空，则左侧显示空态，右侧不渲染任务线

## 4. 数据与实现落点

主要改动落点：

- `Toolbox/Core/API/QuestlineProgress.lua`
  - 新增“枚举当前任务日志任务”的领域 API
  - 统一返回已映射/未映射任务的运行时字段
- `Toolbox/Modules/EncounterJournal.lua`
  - 状态视图左侧改为消费 Quest Log 当前任务 API
  - 调整 `resolveSelectionWithModel` 的 `status` 视图选择恢复逻辑
  - 调整 `render` 中 `status` 视图的右侧渲染逻辑与未映射详情回退
- `tests/logic/spec/encounter_journal_event_lifecycle_spec.lua`
  - 新增状态视图当前任务列表与右侧任务线回归用例
- `tests/logic/spec/questline_progress_spec.lua`
  - 新增 Quest Log 当前任务 API 逻辑测试

不涉及：

- `Toolbox/Core/Foundation/Config.lua` 默认值新增
- `Toolbox.toc`
- 新本地化键（优先复用现有键）

---

## 5. 验收要点

1. 进入 `状态` 视图时，左侧显示“当前任务列表”，不再显示地图树。
2. 左侧列表覆盖角色任务日志中的全部当前任务。
3. 左侧不出现 `pending` 任务。
4. 对于存在任务线映射的任务，点击左侧任务后，右侧立即显示其所属完整任务线。
5. 右侧任务线中当前任务节点高亮。
6. 对于没有任务线映射的当前任务，右侧回退为纯任务详情。
6. `类型 / 地图` 视图行为保持不变。
7. 视图切换与任务切换不引入固定延迟。

---

## 6. 非目标

1. 不恢复或保留状态视图的地图树过滤。
2. 不调整 `类型 / 地图` 视图布局。
3. 不新增“当前任务”独立视图。
4. 不引入新的存档字段或迁移。

---

## 7. 补充决策（2026-04-11）

在现有三视图结构不变的前提下，补充一条样式规则：

1. `地图` 视图左侧任务节点按任务状态染色。
2. `类型` 视图中的任务节点同样按任务状态染色。
3. 当前仅为 `completed` 提供状态色：绿色。
4. `selected` 优先级高于状态色：
   - 若任务已完成但当前被选中，保持现有蓝色高亮
   - 仅在未选中时显示绿色
5. 不改动 `active / pending` 的默认文字颜色。

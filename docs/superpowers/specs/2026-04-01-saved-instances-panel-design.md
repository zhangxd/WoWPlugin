## 1. 目标

- 提供**独立可拖动窗口**（布局参考冒险指南：**左侧列表 + 右侧详情**）。
- **数据两层**：
  - **有锁定进度**：通过 `GetNumSavedInstances` / `GetSavedInstanceInfo` 等与当前角色存档一致。
  - **全量浏览（选项 B）**：除上述外，还能列出**当前角色暂无进度**的地下城/团本条目，行为接近冒险指南的「可看全部分类」，而非仅「有 CD 的条」。
- **Boss 进度**：对有进度的实例，用 `GetSavedInstanceEncounterInfo` 展示各首领击杀情况（以 API 实际提供为准）。
- **筛选**：与冒险指南习惯一致的主分类（如地下城 / 团队副本 / **全部**），并保留「展示全部」语义下的合并展示。
- **跳转**：在能解析 ID 时，**打开世界地图**至对应区域、**打开冒险指南**至副本或首领详情。
- **坐骑掉落**：展示相关首领/副本的坐骑掉落；**是否已拥有**通过 `C_MountJournal`（及当前版本等价 API）标记。
- **成就**：与当前选中副本/内容**相关**且**尚未完成**的成就列表（可跳转至成就面板或手册，以 API 能力为准）。
- **宠物**：与当前内容**相关**且**尚未收集**的宠物（战斗宠物收藏）；数据来源为掉落/来源链路与手册 loot 等，与游戏内「未收集」含义一致。
- **玩具**：与当前内容**相关**且**尚未收集**的玩具（玩具箱）；**是否已收集**以 `C_ToyBox`（及正式服等价 API）为准。
- **美术（配图）**：详情区副本/首领等配图**优先**使用 **`C_EncounterJournal`**（及正式服等价 API）返回的 **贴图路径** 或 **Atlas**，与冒险手册同源资源；无法取得时降级为占位或省略，避免硬编码易过期路径。
- **工程约束**：新功能走 `RegisterModule`；`ToolboxDB.modules.<moduleId>`；玩家可见字符串仅 [Locales.lua](../../Toolbox/Core/Locales.lua)；聊天提示经 **Toolbox.Chat**；窗口拖动与 **Toolbox.Mover** 约定一致；成就/宠物/玩具查询若多处复用，经**领域对外 API**封装，符合 [AGENTS.md](../../AGENTS.md) 领域对外 API 约定。

## 2. 范围说明：选项 B 带来的含义

| 维度 | 仅「有进度」 | **选项 B（已选）** |
|------|----------------|---------------------|
| 左侧列表 | 仅有锁定存档的实例 | **另需**覆盖手册中可见的地下城/团本（无进度时仍可浏览条目） |
| 数据来源 | `GetSavedInstanceInfo` 为主 | **叠加** `C_EncounterJournal`（或当前正式服等价）枚举实例/首领；与存档数据 **合并展示** |
| 复杂度 | 中 | **高**：ID 映射、性能、缓存、无进度时的详情占位 |

**建议实现策略（可在计划中分期）**：

- **第一期（最小可用）**：锁定列表 + 筛选 + Boss 进度 + 地图/手册跳转（在能映射的前提下）+ 坐骑（有 encounter/journal 映射时）。
- **第二期（补全 B）**：无进度条目的完整浏览、与 EJ 分类对齐的筛选、性能与缓存策略。
- **第三期（可选或并入二期）**：成就/宠物/玩具区块的完整数据链与跳转；依赖映射表与缓存时分期交付。

具体分期以实现计划为准；本文档描述**最终形态**，避免遗漏需求。

## 3. 界面结构

- **顶栏**：与冒险指南类似的 **分类筛选**（地下城 / 团队副本 / **全部**）；可选难度或子筛选（实装时按 API 能力裁剪）。
- **左栏**：可滚动列表。每条目至少包含：名称、类型提示、**是否有当前角色进度/锁定**（图标或标签区分「有进度」「无进度」）。
- **右栏**：
  - **配图区（可选）**：实例/当前选中首领的展示图，**优先**用手册 API 返回的 texture/atlas（见 §1 美术）；注意比例与裁剪。
  - 实例概要（名称、难度、重置时间——**仅对有进度/锁定时有意义**；无进度时显示说明或手册摘要字段）。
  - **Boss 列表**（有进度时来自 `GetSavedInstanceEncounterInfo`；无进度时来自手册首领列表或占位）。
  - **坐骑掉落区**：列表 + **已拥有/未拥有**（`C_MountJournal`）。
  - **成就区**：与本副本/区域**相关**且**未完成**的成就（可折叠；条数多时分页或懒加载）。
  - **宠物区**：**未收集**的、来源与当前副本相关的战斗宠物（展示规则与坐骑区类似，数据来源见 §5.1）。
  - **玩具区**：**未收集**的、来源与当前副本相关的玩具（`C_ToyBox` 等）。
  - 操作：**地图**、**冒险指南**（副本级 / 首领级在 API 允许时）；成就/宠物/玩具在可行时提供**跳转游戏内对应界面**的按钮或链接。

### 3.1 与冒险手册的关系（已选：**仅加入口按钮**）

- **主功能界面**：仍为 Toolbox **自有窗口**（本节上表）；列表、筛选、Boss、坐骑、成就、宠物、玩具等**全部在本窗口内**完成，**不**把上述区块嵌入手册内部、**不**改写手册原有 Tab/列表 DOM。
- **冒险手册（EncounterJournal）**：仅增加 **一个（或一组）入口按钮**（文案见 `Locales.lua`），点击后 **打开/前置** Toolbox 面板；逻辑与 **`/toolbox` 子命令** 打开同一窗口一致。
- **可选增强（非必须）**：若正式服 API 允许读取手册当前选中的实例/首领，可在点击入口时 **带参打开** Toolbox 并预选对应条目；若做不到，则仅打开空面板或上次状态。
- **实现注意**：手册 Frame 结构随版本变化，锚点需 **`pcall` + 延迟挂载**（如 `Blizzard_EncounterJournal` 的 `ADDON_LOADED`）；失败时 **静默降级**（不显示按钮），不影响斜杠与设置页入口。
- **设置项（建议）**：`在冒险手册显示 Toolbox 入口` 开关，默认开启；关闭后不在手册上创建按钮。

## 4. 数据与 ID 映射

- **存档侧**：`GetSavedInstanceInfo` / `GetSavedInstanceEncounterInfo`。
- **全量浏览侧**：`C_EncounterJournal`（实例、首领、loot）；具体函数名以正式服为准。
- **统一模型（概念）**：内部使用「展示行」结构，字段可包含：`journalInstanceID`、`encounterID`、`uiMapID`、与存档实例的关联键、是否有锁定等。
- **映射风险**：`instanceIndex`（存档）与 **手册 ID**、**地图 ID** 不一致，需集中封装映射层；无法映射时按钮降级（灰显 + `Toolbox.Chat` 一行说明）。

## 5. 坐骑与收藏状态

- 从手册 loot 或维护的补充表得到「坐骑相关物品 / 坐骑 ID」。
- **是否拥有**：`C_MountJournal` 查询；封装在模块内或薄领域对外 API（若多处复用再抽到 `Core`）。

### 5.1 成就、宠物、玩具（未完成 / 未收集）

| 类型 | 展示语义 | 主要 API 方向（正式服以实装时文档为准） | 与副本的关联方式 |
|------|-----------|----------------------------------------|-------------------|
| 成就 | 相关且**未完成** | `C_AchievementInfo` 等 | 成就分类 / criteria / 地图与 **journalInstanceID** 的映射；手册或成就 UI 可对照；必要时 **小表**维护「实例→成就 ID 列表」 |
| 宠物 | **未收集**战斗宠物 | `C_PetJournal` 等 | 掉落物→物品→宠物 ID；与手册 loot 交叉；无数据时区块为空或简短说明 |
| 玩具 | **未收集** | `C_ToyBox` 等 | 同上，物品→玩具 ID；以客户端「已加入玩具箱」状态为准 |

- **一致性**：展示状态须与游戏内成就、宠物、玩具界面**一致**，避免插件自行一套判定规则。
- **性能**：全量扫成就/宠物库可能昂贵；须 **惰性加载**、按选中实例再查、**缓存**与分帧（见 §8）。
- **领域对外 API**：若实现中出现多处直接调用成就/宠物/玩具底层 API，应抽到 **`Toolbox` 下薄领域对外 API**（命名在实现时定），模块只经领域对外 API 调用，符合 AGENTS「先接口后功能」。

## 6. 模块与持久化

- **新 `moduleId`**（名称在实现时确定，如 `saved_instances` / `instance_browser`）。
- **`ToolboxDB.modules.<id>`**：窗口几何、筛选状态、可选缓存版本号、用户选项（例如是否显示无进度条目默认展开）。
- **拖动**：`Toolbox.Mover.RegisterFrame(..., key, { dragRegion = 标题栏 })`，key 与 `modules.mover.frames` 约定一致，**不**重复存两套坐标。

## 7. 入口

| 入口 | 行为 |
|------|------|
| **`/toolbox` 子命令**（如 `instances` / `cd`） | 打开/前置 Toolbox 面板 |
| **设置页 · 本模块** | 总开关、**打开面板**按钮；可选「在冒险手册显示入口」 |
| **冒险手册 · 仅入口按钮** | 见 §3.1；与独立窗口同一套 UI，不在手册内嵌面板 |

## 8. 风险与验收要点

- **API 变动**：手册与地图打开方式随版本变化，映射与打开逻辑需集中、注释版本假设；**EJ 返回的美术字段名**（texture / atlas）也可能调整，配图逻辑需与数据层一并封装、可降级。
- **手册入口按钮**：锚点失效时仅影响该按钮显示，核心功能仍可通过斜杠/设置打开；需在注释中标明依赖的父 Frame 名便于版本更新。
- **性能**：全量列表 + loot 扫描可能重；成就/宠物/玩具关联查询可能更重，需惰性加载、缓存、分帧。
- **验收**：有/无进度条目均能区分；筛选与「全部」行为符合预期；坐骑标记与游戏内收藏一致；**成就未完成、宠物/玩具未收集**与游戏内状态一致；跳转在可映射时可用，不可映射时有降级而非报错刷屏。

## 11. 方案评估（修订记录）

### 11.1 与 AGENTS.md / README 的契合度

| 要求 | 评估 |
|------|------|
| 独立 `moduleId`、`ToolboxDB.modules.*` | 符合；单一模块可承载，但若单文件过大，宜拆为模块内多文件（数据/UI/领域对外 API 调用分离）。 |
| `Locales.lua`、`Toolbox.Chat`、`Toolbox.Mover` | 符合规格表述；实现时忌在模块内绕过 Chat。 |
| 领域对外 API | 成就/宠物/玩具/坐骑若多处直调 API，**必须先**薄领域对外 API再模块——规格已要求；**增量实现**时建议「用到再建接口」，避免一次性空壳过多。 |
| Core 不堆业务 | 符合，前提是业务留在 `Modules/*` + 领域对外 API；映射与缓存逻辑**勿**塞进 `Bootstrap`。 |
| 更新 Toolbox-addon-design.md | 合并前需补 §2.2、TOC 等（规格 §9 已列）。 |
| README | 上线能力后补一句功能概览，与现有「功能概览」列表风格一致。 |

### 11.2 优势

- **入口策略合理**：独立窗口 + 手册仅按钮，兼顾**可维护性**与**发现性**，避免深度 Hook 手册 DOM。
- **数据聚合**：以「副本/实例」为轴同时看进度、坐骑、成就、宠物、玩具，与玩家心智一致，**差异化**于单独打开收藏界面。
- **分期写法**：规格 §2 已分三期，利于控制爆炸范围；**关键**是实施计划里写死「每期交付物与不做清单」。

### 11.3 主要风险与挑战

| 风险 | 说明 |
|------|------|
| **范围过大** | 「B 全量 + 多收藏维度」接近**半套手册 + 半套收藏追踪**，单人副业易长期拖延；测试面大。 |
| **ID 与映射** | 存档索引、journalInstanceID、uiMapID、成就/掉落链**多处对齐**，边界情况多，需集中「映射层」单测思路（游戏内手测为主）。 |
| **性能** | 全量 EJ 枚举 + loot + 多收藏扫描，若无缓存/分帧，低端机会卡；规格 §8 已提示，**实现时必须落地**。 |
| **与暴雪功能重叠** | 玩家可能问「为何不用游戏自带」——价值在**一屏聚合**与**副本上下文**；若体验不如原生，易被弃用。 |
| **版本维护** | 手册入口锚点、EJ API、美术字段均可能变；需注释与**降级**策略（规格已要求）。 |

### 11.4 路线建议（不修改「最终形态」，只约束落地顺序）

1. **第一期（强烈建议收窄）**：**锁定进度 + 左列表 + 右详情（Boss/概要）+ 地图/手册跳转（可映射时）+ 坐骑（loot 可解析时）+ 配图（EJ 有则显示）+ `/toolbox` 与设置入口**。  
   - **暂不**：选项 B 全量无进度浏览，或仅做**极简**（例如只多显示「手册中同分类名称」占位）。  
   - **暂不**：成就/宠物/玩具整块，或只做 UI 占位一条说明「后续版本」。
2. **第二期**：补全 **B**（无进度浏览）、筛选与缓存、手册入口按钮（若第一期未做）。
3. **第三期**：成就 / 宠物 / 玩具分块上线，每块可再分子版本；领域对外 API随块增量添加。

若坚持**第一期即含 B 与全部收藏**，建议在实现计划中设**硬性里程碑**（如先跑通 1 个团本 + 1 个地下城端到端），否则易陷入全局数据泥潭。

### 11.5 结论

- **方向可行**，与 Toolbox **模块 + 领域对外 API** 模型一致；**风险主要在范围与时间**，不在架构本身。  
- **建议**：以 §11.4 分期为**强约束**写实现计划；规格正文「最终形态」可保留作为产品北极星，**编码顺序**服从分期。  
- 下一步：用户确认是否采纳 **§11.4 的收窄版第一期**；确认后可进入 **writing-plans**（实现计划），再编码。

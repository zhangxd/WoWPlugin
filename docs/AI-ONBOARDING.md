# AI 协作开发：读档顺序与文档整理约定

面向「以后主要用 AI 加功能」时，减少重复说明、避免设计与实现漂移。

---

## 1. 每次加功能时，让 AI 必读（按顺序）

| 顺序 | 文件 | 作用 |
|------|------|------|
| ① | [Toolbox-addon-design.md](./Toolbox-addon-design.md) | **唯一总体方案**：架构、模块契约、数据约定、各能力边界；新功能必须能嵌进这套模型。 |
| ② | 本仓库根目录 `README.md`（若有） | 如何安装、TOC 名、支持的客户端；避免 AI 假设错游戏版本。 |
| ③ | **本次任务** | 用户消息里写清：模块名/行为/验收标准；若复杂，附 `docs/specs/` 下当期规格一节（见下）。 |

**可选、按需打开**

| 情况 | 再读 |
|------|------|
| 改核心加载/DB/注册表 | `Core/` 下实际文件 + 设计文档「2.1 通用架构」 |
| 只加一个模块 | `Modules/` 里**一个**现有模块当模板 + 设计文档「2.2 映射表」 |
| 与暴雪 Frame 名相关 | `docs/specs/` 或模块内注释里的白名单、版本备注 |

**不要**让 AI 先通读整个 `Modules/` 所有文件，除非在做重构；**点名文件**更高效。

---

## 2. 文档怎么分层（建议）

```
docs/
├── Toolbox-addon-design.md   # 总设计：架构 + 约定 + 已有模块说明（随功能迭代更新「映射表」）
├── AI-ONBOARDING.md          # 本文件：给 AI / 协作者的读档顺序
└── specs/                    # 可选：按日期或按功能的短期规格
    └── YYYY-MM-DD-<topic>.md # 一次大需求或复杂交互时写，实现后可合并回总设计或归档
```

- **总设计（Toolbox-addon-design.md）**：长期真相；模块列表、扩展点、`ToolboxDB` 形状、里程碑只维护这一份。
- **当期规格（specs/）**：一次需求对话写不清时，用一篇短文写「输入/输出/边界/验收」；**合并进总设计或删除过期内容**，避免 specs 堆成垃圾场。
- **代码内注释（约束）**：仓库内 Lua **必须有注释**（文件头 + 关键逻辑），**注释一律使用简体中文**（API/Frame 名等可中英混排）。玩家可见文案放入 `Toolbox/Core/Locales.lua`，不要硬编码中文/英文在模块里。

---

## 3. 加新功能时，人要补什么（给 AI 的最小信息包）

在对话里或 `docs/specs/xxx.md` 里尽量包含：

1. **模块 id**（如 `cooldown_overlay`）是否与 [设计文档映射表](./Toolbox-addon-design.md) 已预留一致。  
2. **客户端**：正式服（本仓库默认）。  
3. **数据**：存在 `ToolboxDB.modules.<id>` 哪些键；是否要迁移。  
4. **设置**：是否需要 `RegisterSettings`、大致控件（开关/滑条/下拉）。  
5. **边界**：不做什么（避免 AI 扩大范围）。  
6. **验收**：例如「重载后位置保留」「战斗中不报错」。

---

## 4. 功能做完后，文档谁更新

| 变更 | 更新哪里 |
|------|----------|
| 新模块上线 | `Toolbox-addon-design.md`：鸟瞰图、§2.2 映射表、§3 数据示例、§6 TOC、§7 里程碑 |
| 仅修 bug、未改行为 | 一般不改总设计；除非发现原约定错误，再改文档 |
| 架构级变更 | 先改 `Toolbox-addon-design.md`，再写代码 |

---

## 5. Cursor / 仓库根指针

仓库根已提供 **[AGENTS.md](../AGENTS.md)**，指向总设计与本读档约定；Cursor 会自动读取（若已启用项目规则）。也可在 `.cursor/rules` 里再写一条简短重复，非必须。

---

## 6. 不建议的做法

- 多份「总架构」互不同步（只保留 **Toolbox-addon-design.md** 为总纲）。  
- 把大段 API 教程复制进仓库（链到 [warcraft.wiki.gg](https://warcraft.wiki.gg) 或官方文档即可）。  
- 无日期、无主题的零碎 `notes.md` 长期堆在根目录。

# 设置宿主重构设计

- 文档类型：设计
- 状态：已确认
- 主题：settings-host-redesign
- 适用范围：`Toolbox/UI/SettingsHost.lua` 的设置树结构、叶子页组织方式、默认打开规则，以及 `quest` 额外设置子页并回主页的设置承载调整
- 关联模块：`mover`、`tooltip_anchor`、`encounter_journal`、`quest`、`navigation`、`minimap_button`、`chat_notify`
- 关联文档：
  - `docs/Toolbox-addon-design.md`
  - `docs/features/quest-features.md`
  - `docs/features/navigation-features.md`
  - `docs/features/encounter-journal-features.md`
  - `docs/designs/quest-design.md`
  - `docs/specs/settings-overview-spec.md`
- 最后更新：2026-04-29

> 本文记录 2026-04-29 用户确认的设置界面整体重构方向。该方案当前只覆盖设置信息架构、页面职责与宿主实现模型，不包含业务功能新增，也不直接修改模块行为语义。

## 1. 背景

- 当前 `SettingsHost` 以“总览页 + 各模块子页面 + 关于页”为主结构，左侧列表基本按模块平铺。
- 当前右侧页面普遍复用同一套公共骨架：简介、启用、调试、清理并重建，再追加模块专属设置区，导致页面观感接近模板复制，重点不够突出。
- 当前设置系统同时存在总览页、通用说明、模块页和关于页，容易出现职责重叠，用户进入后需要自己判断“先看哪里、哪里才是真正能改的地方”。
- 用户本轮明确目标不是新增能力，而是重做设置组织方式与阅读路径，重点解决“视觉和排版太旧、读起来累”的问题，同时保持暴雪原生设置风格。

## 2. 设计目标

- 保留左侧设置树，并让左侧叶子节点直接对应最终设置页，不再引入总览页或组内再导航。
- 将右侧页面从“统一模板”调整为“按页面职责组织内容”，降低重复说明与低频控件的视觉权重。
- 以玩家任务视角重新组织左侧叶子页，使页面名称本身就是玩家可理解的分类。
- 保持现有模块行为、`moduleId` 与 `ToolboxDB.modules.*` 数据落点不变，把改动集中在设置宿主与设置内容承载方式。
- 统一 `/toolbox`、ESC 菜单、小地图按钮三种设置入口的默认打开规则。

## 3. 非目标

- 不新增模块、不新增玩家可见入口、不修改 `Toolbox/Toolbox.toc`。
- 不改变各模块现有运行时功能行为语义，只重排设置组织和呈现方式。
- 不新增新的全局存档结构或迁移规则；现有 `ToolboxDB.modules.*` 落点继续沿用。
- 不借本轮重构扩展新的视觉皮肤、仪表盘、收藏项、最近使用摘要等偏“插件控制台”的能力。
- 不在右侧页面内再做一层“分组首页”或“组内导航”。

## 4. 方案对比

### 4.1 方案 A：纯叶子页

- 做法：
  - 左侧仅保留一个根节点 `Toolbox`。
  - 根节点下直接挂 6 个叶子页：`界面整理`、`冒险手册`、`任务与进度`、`导航`、`通用与入口`、`关于`。
  - 点击任一叶子页，右侧直接进入该页完整设置内容。
- 优点：
  - 最符合用户确认的“左侧树 + 叶子页直达设置页”目标。
  - 页面职责清晰，不再需要额外首页承接说明。
  - 有利于把总览页、关于页和通用项的职责彻底拆开。
- 风险 / 缺点：
  - 需要重新设计默认打开落点，因为旧的总览页不再存在。
  - `quest` 现有 Quest Inspector 额外子页需要并回主页承载。

### 4.2 方案 B：根页 + 叶子页

- 做法：
  - 保留可点击的 `Toolbox` 首页，再在左侧挂各叶子页。
  - 首页承担欢迎说明、入口说明或少量全局项。
- 优点：
  - 保留“进入设置先看到一个首页”的传统路径，迁移成本较低。
  - 某些全局说明有单独落点。
- 风险 / 缺点：
  - 首页极易与“通用与入口”或“关于”发生职责重叠。
  - 页面层级变多，削弱用户希望的“点叶子直接进设置”的直达感。

### 4.3 方案 C：纯叶子页 + 页内折叠块

- 做法：
  - 左侧仍然采用纯叶子页结构。
  - 右侧页面内部使用较多折叠区组织设置内容。
- 优点：
  - 为后续扩展预留空间。
  - 在单页内容较长时能部分缓解滚动压力。
- 风险 / 缺点：
  - 若使用不当，容易让页面重新长成“页面里再分组”的结构。
  - 会削弱“贴近暴雪原生”的目标，增加阅读与交互成本。

### 4.4 选型结论

- 选定方案：方案 A。
- 选择原因：
  - 方案 A 与用户明确表达的结构目标完全一致：左侧树保留、叶子节点就是最终设置页、右侧只做内容重排。
  - 方案 B 仍会保留一个容易打架的首页。
  - 方案 C 虽然可扩展，但会重新引入额外分层，不符合本轮“读起来更轻”的主诉求。

## 5. 选定方案

### 5.1 左侧树结构

左侧树只保留一个根节点与 6 个叶子页：

- `Toolbox`
- `界面整理`
- `冒险手册`
- `任务与进度`
- `导航`
- `通用与入口`
- `关于`

约束如下：

- `Toolbox` 仅作为树根，不再承载可点击的总览页。
- 左侧叶子页是唯一的设置主入口，不再保留总览页或组内再导航。
- 右侧页面不再以“模块页”为唯一组织单位，而改为由宿主按叶子页职责组合内容。

### 5.2 6 个叶子页职责

#### 界面整理

- 组合模块：`mover` + `tooltip_anchor`
- 页面目标：集中放置直接影响界面摆放、拖动与 tooltip 显示方式的设置项。
- 建议内容顺序：
  - 页面主开关与最常改项
  - 窗口拖动相关项：拖动命中区域、战斗中是否允许拖动
  - Tooltip 锚点模式与偏移
  - 页面底部再放 `debug` 与 `重置/重建`

#### 冒险手册

- 组合模块：`encounter_journal`
- 页面目标：仅承载冒险手册增强相关设置。
- 建议内容顺序：
  - 主开关
  - 图钉常驻显示等当前有效设置项
  - 与当前行为直接相关的简短说明
  - 页面底部的 `debug` 与 `重置/重建`

#### 任务与进度

- 组合模块：`quest`
- 页面目标：承载任务模块的所有可配置项，并取消额外设置子页。
- 建议内容顺序：
  - 主开关
  - 打开任务窗口的入口按钮
  - 任务视图开关
  - 最近完成任务数量上限
  - Quest Inspector 输入、查询与结果区域并回本页下半部分
  - 页面底部的 `debug` 与 `重置/重建`

明确调整：

- 不再保留左侧单独的 Quest Inspector 设置子页。
- `Quest Inspector` 继续作为 `quest` 模块的一部分存在，但改为“任务与进度”叶子页中的低频工具区。

#### 导航

- 组合模块：`navigation`
- 页面目标：承载导航模块当前与后续设置项。
- 建议内容顺序：
  - 主开关
  - 当前导航能力说明
  - 后续新增设置沿“高频先、说明后”的顺序扩展
  - 页面底部的 `debug` 与 `重置/重建`

#### 通用与入口

- 组合内容：
  - 宿主自带的语言设置区
  - 宿主自带的重载界面区
  - `minimap_button`
  - `chat_notify`
- 页面目标：集中放置插件全局项与入口行为项，不再由总览页承接。
- 建议内容顺序：
  - 语言设置
  - 重载界面按钮
  - 小地图按钮显示与位置相关项
  - 小地图坐标显示与挂点
  - 小地图飞出方向、形状、预览与功能池
  - 聊天加载提示、复制入口、颜色设置
  - 页面底部的低频动作区

#### 关于

- 组合内容：纯宿主静态说明，不承载可操作设置。
- 页面目标：收口版本、客户端、入口说明与插件范围说明。
- 建议内容顺序：
  - 插件一句话说明
  - 版本
  - 客户端
  - 常用打开方式
  - 插件覆盖范围

明确约束：

- `关于` 不再承载语言、重载、小地图按钮或聊天提示等可操作设置。
- `关于` 与 `通用与入口` 彻底分工：前者静态说明，后者全局设置。

### 5.3 右侧页面统一骨架

所有叶子页遵循同一视觉骨架，但不再强制复用旧的公共模板顺序：

1. 页标题 + 一句话说明
2. 主开关区
3. 设置内容区
4. 次要动作区（`debug`、`重置/重建`）

补充约束：

- 高频项靠上，低频项靠下。
- 解释文字尽量贴近对应设置项，不单独堆成长段说明。
- 不再让 `debug` 与 `重置/重建` 占据页面首屏注意力。
- 不再要求每一页都按“简介 -> 启用 -> 调试 -> 清理并重建 -> 模块内容”的固定模板渲染。

### 5.4 默认打开规则

以下 3 个入口统一使用同一打开规则：

- `/toolbox`
- `ESC -> 工具箱`
- 小地图按钮打开设置

规则如下：

- 优先打开“上次停留的叶子页”。
- 若首次打开、历史记录缺失或记录失效，则回退到 `通用与入口`。

选择原因：

- 本轮已移除总览页作为默认落点。
- 用户确认设置界面的主要使用场景是“平时经常进来改少数几项”。
- `通用与入口` 作为首次落点最接近“插件设置入口页”的角色。

### 5.5 宿主实现模型

本轮只重构设置宿主的组织方式，不改变模块运行时行为：

- 不改 `moduleId`
- 不改 `ToolboxDB.modules.*` 数据落点
- 不改功能开关语义
- 改动集中在 `Toolbox/UI/SettingsHost.lua`

建议实现方式：

- 由宿主显式声明 6 个叶子页，而不是简单按 `RegisterSettings` 结果平铺模块页面。
- 将旧的“模块公共模板”拆成可组合片段，由叶子页自行决定摆放顺序。
- Quest Inspector 不再通过 `GetSettingsPages()` 额外注册左侧子页，而是改为供 `任务与进度` 页面调用的内容块构建函数。
- 原总览页中的 `BuildLanguageSection`、`BuildReloadSection` 迁移到 `通用与入口` 页。
- `minimap_button` 与 `chat_notify` 不再各占一个左侧叶子位，而是作为 `通用与入口` 内部内容块组合。

## 6. 影响面

- 数据与存档：
  - 现有 `ToolboxDB.modules.*` 保持不变。
  - 允许新增“上次停留叶子页”这类设置宿主级全局状态，但应优先放入现有全局配置区域，而不是创建新模块存档。
- API 与模块边界：
  - 模块的业务行为与对外 API 不变。
  - `quest` 的 Quest Inspector 设置子页承载方式改变，但 Quest Inspector 能力本身不改变归属。
  - `SettingsHost` 从“模块页宿主”提升为“叶子页组合宿主”。
- 文件与目录：
  - 核心改动集中在 `Toolbox/UI/SettingsHost.lua`
  - 预计需要少量调整：
    - `Toolbox/Modules/Quest.lua`
    - `Toolbox/Modules/MinimapButton.lua`
    - `Toolbox/Modules/ChatNotify.lua`
    - `Toolbox/Core/Foundation/Locales.lua`
- 文档回写：
  - 方案落地后，必须更新 `docs/Toolbox-addon-design.md` 中关于 `SettingsHost`、页面结构、Quest Inspector 承载位置和模块映射的描述。
  - `quest-features.md` 中关于 Quest Inspector 为“独立设置子页面”的描述需要改写。
  - `FEATURES.md` 中关于“每个模块都有独立设置页”的表述需要同步校正。

## 7. 风险与回退

- 风险：
  - `SettingsHost` 的页面注册方式会从当前“模块平铺”转向“宿主显式组合”，若拆分不清晰，容易在重构中引入页面内容丢失或顺序错乱。
  - Quest Inspector 从独立设置子页并回主页后，若布局处理不当，可能让“任务与进度”页过长。
  - `通用与入口` 会成为内容最多的叶子页，若内容顺序安排不当，仍可能读起来累。
  - 战斗内独立宿主模式必须继续支持直接打开指定叶子页，不能因为移除总览页而退化。
- 回退或缓解方式：
  - 先重写左侧叶子树注册逻辑，再逐步将页面内容组合迁入新叶子页。
  - 在 Quest Inspector 并回主页前，保留旧内容构建函数，优先改“承载位置”而非改“查询逻辑”。
  - 实现阶段按“左侧树 -> 页面组合 -> Quest Inspector 并回 -> 删除旧总览与旧子页注册”的顺序推进，便于局部回退。
  - 若整体验证不通过，可先回退到“总览页 + 模块页”的旧注册模型，再逐步重做页面内部排版。

## 8. 验证策略

实现落地后应至少验证以下结果：

1. 左侧树只保留一个根 `Toolbox` 与 6 个叶子页。
2. 点击任一叶子页，右侧直接进入最终设置内容，不再出现总览页或组内再导航。
3. `任务与进度` 页内直接包含 Quest Inspector，不再保留独立设置子页。
4. `通用与入口` 至少收编语言、重载、小地图按钮与聊天加载提示。
5. `关于` 仅保留静态说明，不再承载可操作设置。
6. `/toolbox`、ESC、小地图按钮三种入口行为一致，并遵循“上次停留页优先，否则回退到通用与入口”的默认规则。
7. 战斗内独立宿主模式仍可直接打开指定叶子页。
8. 语言切换后，所有叶子页标题、说明与控件文案都能完整重建。
9. `debug` 与 `重置/重建` 在各叶子页中都被压到底部，不再占据页面首屏。
10. 模块行为、`moduleId` 和 `ToolboxDB.modules.*` 数据落点不发生语义变化。

建议验证方式：

- 静态校验：补充或更新设置页结构相关测试，锁定左侧树和子页面注册口径。
- 逻辑校验：验证 Quest Inspector 并回主页后，查询逻辑与结果展示不退化。
- 游戏内验证：检查三种设置入口、叶子页切换、语言切换、战斗内独立宿主与小地图按钮入口。

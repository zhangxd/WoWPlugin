## 1. 背景

- `encounter_journal` 当前已经承载多项冒险指南增强能力，但历史记录分散在多份阶段性文档中。
- 后续还会继续在冒险指南相关功能上新增能力，因此需要一份“当前已实现基线”的需求文档，作为未来增量需求的对照面。
- 本文档以当前仓库代码为准，记录现阶段已经实现并对外可见的功能边界，而不是未来规划。

## 2. 目标

- 明确 `encounter_journal` 当前已经交付的能力范围。
- 明确哪些能力虽然通过其他模块展示，但仍应计入冒险指南整体能力。
- 为后续新增功能提供历史基线与验收对照。

## 3. 范围

### 3.1 In Scope

- 副本列表“仅坐骑”筛选。
- 副本列表锁定信息叠加与悬停详情。
- 副本详情页“仅坐骑”筛选。
- 副本详情页当前难度重置标签。
- 左侧资料片树导航。
- 资料片下的 `地图任务线 / 任务类型` 两个浏览入口。
- 地图主区任务线单展开列表。
- 任务列表 tooltip 与点击详情弹框。
- 任务线名称运行时解析与 `QuestLine #<id>` 回退。
- 任务类型运行时名称解析与 `Unknown Type (%s)` 回退。
- 任务页签左树状态记忆。
- 冒险指南主页页签顺序与显隐设置。
- 小地图“冒险手册”入口与锁定摘要联动。
- `EJMicroButton` tooltip 锁定摘要联动。

### 3.2 Out of Scope

- Tooltip 锚点增强。
- 窗口拖动能力。
- 统一聊天输出本身。
- 尚未落地的未来冒险指南功能。
- 面向开发者的 API 接入说明。

## 4. 已确认决策

- 主归属模块为 `encounter_journal`。
- 小地图“冒险手册”入口由 `minimap_button` 呈现，但其功能定义仍计入冒险指南整体能力。
- 锁定与坐骑掉落相关数据由 `Toolbox.EJ` 提供，任务模型与运行时字段由 `Toolbox.Questlines` 提供。
- 任务页签导航固定为“左侧资料片树 + 资料片下 `地图任务线 / 任务类型` 双入口 + 地图主区任务线单展开列表”。
- 任务线名称优先使用运行时 API；静态导出不再保留结构化 `Name_lang` 字段，运行时失败时统一回退为 `QuestLine #<id>`。
- 任务类型名称优先使用运行时 `GetQuestTagInfo / Enum.QuestTagType` 结果；拿不到名称时统一回退为 `Unknown Type (%s)`。
- `encounter_journal` 后续子专题需求不再拆独立规格文档，统一续写本文件。
- 当前需求文档记录的是“已实现基线”，后续新增功能应以本文件为历史对照，而不是覆盖掉现有实现边界。

## 5. 待确认项

- 无。本文件只描述当前已实现内容，不新增待决设计选项。

## 6. 验收标准

1. 在地下城 / 团队副本列表中，用户可以启用“仅坐骑”筛选并看到过滤结果。
2. 在副本列表中，用户可以直接看到当前角色的锁定重置时间；团队副本同时看到进度。
3. 鼠标悬停副本列表项时，用户可以看到更完整的锁定详情。
4. 在副本详情页中，用户可以切换“仅坐骑”并看到当前副本掉落中的坐骑物品。
5. 在副本详情页中，用户可以看到当前所选难度的重置标签。
6. 在冒险指南根页签中，用户可以进入“任务”页签，并通过左侧资料片树进入 `地图任务线` 或 `任务类型` 两种浏览路径。
7. 地图路径下，选中地图后主区显示任务线单行列表，点击任务线原地展开任务列表，再次点击折叠。
8. 类型路径下，选中类型大类后主区直接显示任务列表。
9. 任务列表只显示任务名称；悬停显示 tooltip；点击后显示任务详情弹框。
10. 若任务具备任务线归属，详情弹框可以回跳到对应地图 / 任务线。
11. 任务页签会记住当前资料片、当前模式、当前地图 / 类型大类和当前展开任务线。
12. 设置页中可调整冒险指南根页签顺序与显隐，并即时生效。
13. 小地图“冒险手册”入口和 `EJMicroButton` tooltip 都能显示当前副本锁定摘要。
14. 当运行时 API 能返回任务线名称时，任务页签中的任务线显示文本使用运行时名称；当运行时 API 不可用或无法解析时，界面稳定回退为 `QuestLine #<id>`。
15. 当运行时 API 能返回任务类型名称时，类型视图使用运行时名称；返回空值时稳定回退为 `Unknown Type (%s)`，界面不报错。
16. `Toolbox/Data/InstanceQuestlines.lua` 的 `questLines` 块不再存在结构化 `Name_lang` 字段，只保留 Lua 注释用于排查。

## 12. 2026-04-15 任务详情查询页面确认

- 决策状态：已确认，可执行。
- 用户确认：
  - 归属模块：`encounter_journal`
  - 查询主键：`QuestID`
  - 数据来源：仅运行时 API
  - 页面形态：在设置中新增独立子页面，不塞回现有 `encounter_journal` 设置页
  - 结果联查：除任务本身字段外，同时查询并展示与该任务强相关的任务线信息
- 实施范围：
  - 在设置中增加一个“任务详情查询”页面。
  - 页面提供 QuestID 输入框、查询按钮和下方结果展示区。
  - 结果展示按“字段名: 字段值”逐行输出，尽可能详细。
  - 结果文本必须可复制。
  - 当任务缓存未就绪时，允许先发起异步加载，再在加载结果返回后刷新结果区。
- 边界约束：
  - 不新增 `RegisterModule`。
  - 不新增新的 slash 命令。
  - 不新增 `Toolbox/Toolbox.toc` 行。
  - 不引入 `Toolbox/Data/*` 静态数据作为补齐来源。
  - 不把结果仅输出到聊天框，设置页必须有可复制的可见结果区。
- 存档约定：
  - 允许在 `ToolboxDB.modules.encounter_journal` 下新增该页面所需的独占字段，例如最近一次查询的 QuestID。
  - 新字段必须在 `Config.lua defaults` 中声明，并做类型归一或迁移。
- 验收标准补充：
  1. 在设置中能看到 `encounter_journal` 下新增的“任务详情查询”子页面。
  2. 输入合法 QuestID 并点击查询后，下方能显示该任务的详细字段文本。
  3. 展示内容至少覆盖任务基础状态、日志状态、任务标签、地图信息、目标列表，以及运行时可解析到的任务线信息。
  4. 结果区中的文本可选择、可复制。
  5. 对非法 QuestID、无返回值或异步加载失败场景，页面给出明确提示且不报 Lua 错。

## 13. 2026-04-15 任务能力拆分为独立模块（quest）确认

- 决策状态：已确认，可执行（用户已明确回复“开动”）。
- 用户确认摘要：
  - 新增独立模块：`moduleId = quest`。
  - `encounter_journal` 完全移除任务相关功能（任务页签、任务设置、任务查询页）。
  - 小地图飞出菜单新增“任务”入口，用于打开 `quest` 模块界面。
  - 底部分页签仅保留两个，且顺序固定：`active_log`（当前任务）在前，`map_questline`（任务线）在后。
  - `map_questline` 继续使用“资料片 -> 混合列表”视图。
  - `quest_type` 入口和视图不再保留。
  - `questInspector`（任务详情查询）迁移至 `quest` 模块。
  - `ToolboxDB.modules.encounter_journal.quest*` 旧键完全移除，不做迁移保留。
- Quest 界面布局约束：
  - 顶部为导航独立行；视觉上与主视图融为一体，但不属于视图内容。
  - 导航行不随主视图滚动。
  - 中部为当前页签主视图区，底部为页签栏。
- 导航组件约束：
  - 导航是独立通用能力，由当前页签视图提供路径节点数据。
  - `active_log` 页签不注册路径节点（导航行保留但无路径节点）。
  - `map_questline` 页签注册可点击路径节点，用于视图定位跳转。
  - 当前节点作为末尾“当前项”展示，不可点击。
- `active_log` 页签约束（样式与信息组织）：
  - 样式复刻系统任务界面（不嵌入 Blizzard 原生任务窗口）。
  - 当前任务区与历史完成区在同页签内展示。
  - 历史完成区位于当前任务页签底部，且默认展开。
  - 任务线页签不再展示当前任务列表。

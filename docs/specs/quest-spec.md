# 任务模块需求规格

- 文档类型：需求
- 状态：可执行
- 主题：quest
- 适用范围：`quest` 当前已实现的独立任务界面、任务浏览与 Quest Inspector 基线
- 关联模块：`quest`、`minimap_button`
- 关联文档：
  - `docs/features/quest-features.md`
  - `docs/designs/quest-design.md`
  - `docs/tests/quest-test.md`
  - `docs/specs/encounter-journal-spec.md`
- 最后更新：2026-04-20

## 1. 背景

- 当前代码已经把原先挂在冒险指南里的任务能力拆成独立 `quest` 模块，但仓库内还没有与之对应的需求基线文档。
- 需要一份按当前实现整理的规格文档，明确 `quest` 模块的边界、入口和验收口径。

## 2. 目标

- 明确 `quest` 当前已经交付的能力范围。
- 明确 `quest` 与 `encounter_journal` 的职责分界。
- 为后续继续扩展独立任务界面时提供稳定的历史对照面。

## 3. 范围

### 3.1 In Scope

- 独立任务界面 `ToolboxQuestFrame`。
- 底部两个视图页签：`active_log` 当前任务视图、`map_questline` 任务线浏览视图。
- 左上角通用导航路径与可点击祖先回退。
- `map_questline` 视图内部的资料片 / 地图 / 任务线节点导航。
- 搜索框过滤任务线与任务名称。
- 最近完成任务记录与展示。
- 任务列表内展开详情与行内跳转动作。
- `quest` 设置页中的 Quest Inspector 子页面。
- 小地图飞出菜单“任务”入口。

### 3.2 Out of Scope

- 冒险指南副本列表、详情页和锁定摘要增强。
- `quest_type` 视图。
- 冒险指南根页签顺序与显隐管理。
- 额外 slash 命令、额外菜单按钮或额外 TOC 行之外的隐藏入口。
- 任务数据导出链路本身的设计与实现。

## 4. 已确认决策

- 主归属模块为 `quest`。
- 主界面宿主为独立 `ToolboxQuestFrame`，不再依附 `EncounterJournal` 根框体。
- 主界面采用不透明底板，并提供独立标题栏；视觉层次参考冒险指南的“标题栏 + 主体内容区”结构。
- `quest` 宿主框标题栏点击后可拖动，行为统一接入 `Toolbox.Mover.RegisterFrame()` 的自建窗体路径，而不是在 `quest` 模块里单独维护一套拖动实现。
- 底部固定保留两个视图页签：`active_log` 与 `map_questline`；默认打开逻辑保持现状，不额外改默认入口。
- `active_log` 视图不再复用左树 + 右区布局，而是改为上下两段：上方当前任务固定显示并可内部滚动，下方历史完成列表可折叠；折叠后上方区域占满。
- 每个视图都可以注册自己的导航节点；`active_log` 至少注册当前视图根节点，供左上角通用导航路径使用。
- `map_questline` 视图继续使用左侧导航 + 右侧主区，但左侧不再提供“当前任务”入口。
- `map_questline` 视图左树中的资料片分组仅显示资料片名称，不显示编号前缀；该规则仅作用于任务界面显示层，不改静态数据中的原始 `ExpansionID`。
- `quest` 内部导航改为“通用节点 + 当前选中节点 + 导航路径”模型；节点只描述父子关系与上下文，不承载页面跳转回调。
- 左上角导航路径显示当前选中节点的祖先链，且祖先节点支持点击回退。
- `map_questline` 视图顶部路径导航改为接入 Blizzard `NavBar` 组件，视觉对齐冒险指南地下城页签中“选中地下城页签后进入副本节点”的原生路径条，而不是继续使用自绘按钮条。
- 顶部路径导航不再放在正文内容框内部，而是移动到宿主标题栏与正文之间、与宿主标题区背景融合的独立头部带；导航栏位于宿主头像 / 标题图标区右侧，右端继续避让搜索框，并在该头部带内垂直居中放大占满。
- 搜索框只保留一层可见输入框边界；外层布局容器不得再额外绘制第二层边框，避免出现双框重叠。
- 任务导航、任务列表、任务详情与 Quest Inspector 数据统一通过 `Toolbox.Questlines` 领域 API 获取。
- 任务列表中的任务名与任务线名均采用“运行时 API 优先、静态数据回退”的口径；任务名优先用 `C_QuestLog.GetTitleForQuestID`，任务线名优先用 `C_QuestLine.GetQuestLineInfo` 返回的 `questLineName`。
- 行内展开详情中的“类型”显示统一采用“类型名字（ID）”格式；名字优先使用现有类型映射解析，若类型名无法解析则回退为默认文案“普通任务”（`EJ_QUEST_TYPE_DEFAULT`），并继续保留 `typeID` 展示。
- 最近完成任务通过 `QUEST_TURNED_IN` 事件维护，保存在 `ToolboxDB.modules.quest.questRecentCompletedList`。
- Quest Inspector 归属 `quest` 模块设置页，不再属于 `encounter_journal`。
- 小地图飞出菜单中的“任务”入口用于打开 `quest` 主界面；打不开时回退到 `quest` 设置页。
- 通用节点导航本轮仅落在 `quest` 模块内部，不上提到 `Core` 级通用层。
- 点击任务后不再显示 tooltip，也不再向聊天框输出任务详情；详情改为在当前列表行内展开，再次点击同一任务时收起。
- 原“详情弹框”退出主交互路径；若存在跳转到任务线或切换到“当前任务”视图的动作，统一放入列表内展开区。

## 5. 待确认项

- 无。本文件只描述当前已实现内容，不新增待决设计选项。

## 6. 验收标准

1. 小地图飞出菜单中的“任务”入口可以打开独立 `quest` 主界面。
2. 设置页“任务”模块中可以点击按钮打开同一主界面。
3. 主界面底部固定显示 `当前任务` 与 `任务线` 两个视图页签，并可相互切换。
4. 主界面标题栏可拖动，拖动命中区位于宿主标题栏本身，而不是正文内容区。
5. `当前任务` 视图不显示左树导航，而是显示上下两段内容：上方为当前任务列表，下方为历史完成列表。
6. 当前任务列表固定可见，每个任务一行；列表超出可滚动。
7. 历史完成列表条数受 `questRecentCompletedMax` 控制，可折叠；折叠后当前任务列表占满内容区。
8. `任务线` 视图继续提供左侧层级导航与右侧内容区，但左侧不再混入“当前任务”入口。
9. `任务线` 视图左上角显示通用导航路径，路径中的祖先节点可点击回退。
10. `任务线` 视图顶部路径导航使用 Blizzard `NavBar` 同类样式，末级节点表示当前位置，祖先节点可点击回退。
11. `任务线` 视图顶部路径导航显示在宿主标题栏下方、正文内容上方的独立头部带中，并与宿主标题区背景融合，而不是继续占用正文内容区第一行。
12. `任务线` 视图顶部路径导航位于宿主头像 / 标题图标区右侧，不会被左上角头像 / 图标盖住，且不会与右上角搜索框重叠；导航按钮在该头部带内垂直居中并放大占满可用高度，导航栏整体宽度不得越过搜索框左边界。
13. 顶部路径导航文本按可见区域左对齐显示，超出可用宽度时也不得继续盖住搜索框。
14. `任务线` 视图左侧层级由 `quest` 内部注册的通用导航节点树驱动，点击节点后右侧按当前选中节点上下文刷新。
15. 点击任务线后会在主区展开对应任务列表，再次点击可折叠。
16. 搜索框只显示单层输入框边界，不出现“外层额外一圈框”和内层可编辑区重叠的双框效果。
17. 搜索框可以按当前视图支持的任务线名或任务名过滤内容。
18. `任务线` 主视图中每条任务线右侧不再显示“下一步：xxx”长文案；仅在任务线已完成时保留短状态“已完成”，未完成时右侧状态区留空。
19. `任务线` 主视图中的任务线行改为两行高度。
20. 任务线展开后的任务行改回单行高度，并且不再显示任务线名称副标题。
21. 行内展开详情中移除“在进行中视图查看”按钮，仅保留跳转到对应地图 / 任务线的动作。
22. 任务列表中的任务项显示任务名；任务名优先使用运行时 API 值，取不到时回退静态名称或任务 ID 兜底。
23. 点击任务后不会显示 tooltip，也不会向聊天框输出调试信息；当前列表中会直接展开该任务的详细信息，再次点击同一任务会收起。
24. 行内展开详情中的“类型”显示为“类型名字（ID）”；当名字无法解析时，类型名字回退为“普通任务”，并保留原始 `typeID` 作为兜底。
25. 在 Quest Inspector 子页面输入合法 `QuestID` 后，可看到可复制的结果文本；若任务缓存未就绪，页面会在异步加载完成后刷新。
26. 当前实现中不再暴露 `quest_type` 视图，也不再依赖 `encounter_journal` 作为任务浏览入口。
27. `任务线` 视图左树与路径中的资料片分组仅显示资料片名称，不显示编号前缀；仅界面显示变化，底层 `ExpansionID` 数据不做改写。

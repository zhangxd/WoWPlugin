# 设置总览页移除效果预览需求

- 文档类型：需求
- 状态：已完成
- 主题：settings-overview
- 适用范围：`Toolbox/UI/SettingsHost.lua` 设置总览页中的“效果预览”整块移除
- 关联模块：无
- 关联文档：
  - `docs/plans/settings-overview-plan.md`
  - `docs/Toolbox-addon-design.md`
- 最后更新：2026-04-29

## 1. 背景

- 当前 `/toolbox` 设置总览页会在模块列表下方渲染一整块“效果预览”区域。
- 该区域包含“提示框锚点”示例按钮与“窗口拖动”说明，属于主页上的额外说明性内容，而不是模块子页本身的设置项。
- 用户已明确要求：从设置主页中移除这整块预览区域，不保留局部残片。

## 2. 目标

- 打开 Toolbox 设置总览页时，不再显示“效果预览”整块。
- 保持 `tooltip_anchor` 模块子页、`mover` 模块子页与其它设置子页不变。
- 清理因该预览区存在而产生的死代码与无用本地化文案。

## 3. 范围

### 3.1 In Scope

- 移除 `Toolbox/UI/SettingsHost.lua` 总览页中对“效果预览”区域的构建与调用。
- 删除仅供该预览区使用的本地化键。
- 补充自动化验证，锁定“设置总览页不再包含效果预览”的口径。

### 3.2 Out of Scope

- 不修改 `tooltip_anchor` 模块的锚点行为、存档结构或模块子页设置项。
- 不修改 `mover` 模块的行为、存档结构或模块子页设置项。
- 不新增模块、不新增玩家可见入口、不修改 `Toolbox/Toolbox.toc`。

## 4. 已确认决策

- 变更范围限定在设置主页总览页，不改 `tooltip_anchor` 模块行为。
- 用户选定方案为：移除整块“效果预览”，而不是只删提示框示例按钮。
- 实现方式选定为：删除总览页调用、删除对应构建函数，并清理不再使用的本地化键。
- 本次改动不涉及 `ToolboxDB` 新键、迁移或模块归属调整。

## 5. 待确认项

- 无。用户已确认移除范围与实现方向，可进入执行。

## 6. 验收标准

1. 打开 `/toolbox` 设置总览页后，不再出现“效果预览”标题、提示框示例按钮或窗口拖动说明。
2. 左侧模块子页仍可正常打开，`tooltip_anchor` 与 `mover` 的独立设置页不受影响。
3. 仓库中不再保留仅供该预览区使用的 `SETTINGS_PREVIEW_*` 本地化键与构建函数残留。
4. 与本次改动相关的自动化测试会先在旧实现上失败，再在移除完成后通过。

## 7. 实施状态

- 当前状态：已完成
- 下一步：无。本次设置总览页预览区已移除并完成自动化验证。

# 设置宿主重构需求

- 文档类型：需求
- 状态：已完成
- 主题：settings-host-redesign
- 适用范围：`Toolbox/UI/SettingsHost.lua` 的设置树结构、叶子页承载方式、默认打开规则，以及 `quest` 的 Quest Inspector 设置承载位置调整
- 关联模块：`mover`、`tooltip_anchor`、`navigation`、`quest`、`encounter_journal`、`minimap_button`、`chat_notify`
- 关联文档：
  - `docs/designs/settings-host-redesign-design.md`
  - `docs/Toolbox-addon-design.md`
  - `docs/features/quest-features.md`
- 最后更新：2026-04-29

## 1. 背景

- 当前 `SettingsHost` 仍以“总览页 + 模块页 + 关于页”的旧组织方式为基础，右侧内容大量复用统一模板，阅读路径偏重“先看宿主，再进模块”。
- 2026-04-29 已完成“设置总览页移除效果预览”这一步收口，但这只是删除旧首页残留，并未解决左侧树层级、页面职责重复和默认打开规则不统一的问题。
- 用户已在设计阶段明确，本轮目标不是给模块加新能力，而是重做设置组织方式，让左侧叶子页直接对应最终设置页，同时保持暴雪原生设置风格与现有模块行为不变。

## 2. 目标

- 将设置界面重构为“一个根节点 + 6 个叶子页”的结构，取消总览页和组内再导航。
- 让 `/toolbox`、ESC 菜单按钮和小地图按钮三种入口遵循同一默认打开规则。
- 按页面职责重排设置内容，把 Quest Inspector 并回“任务”页，并降低 `debug`、`重置/重建` 等低频动作的视觉权重。
- 保持现有 `moduleId`、`ToolboxDB.modules.*` 数据落点和运行时行为语义不变。

## 3. 范围

### 3.1 In Scope

- 重写 `Toolbox/UI/SettingsHost.lua` 的页面注册模型，从“按模块平铺”改为“宿主显式声明叶子页并组合内容块”。
- 将左侧树收敛为 6 个叶子页：`通用`、`界面`、`地图`、`任务`、`冒险手册`、`关于`。
- 将 `Quest Inspector` 从 `quest` 模块额外设置子页并回“任务”页内部，取消左侧独立入口。
- 将语言设置、重载界面、小地图按钮和聊天提示等内容收编到“通用”页。
- 收口“通用”页中的 `minimap_button` 设置项：固定圆形主按钮与横向悬停菜单，移除预览与拖放式功能池，改为勾选加入悬停菜单。
- 统一三种设置入口的默认打开规则，并在需要时新增宿主级全局状态以记录“上次停留叶子页”。
- 更新与设置结构直接相关的本地化、静态校验与设计/功能文档描述。

### 3.2 Out of Scope

- 不新增模块、不新增玩家可见入口、不修改 `Toolbox/Toolbox.toc`。
- 不修改 `mover`、`tooltip_anchor`、`navigation`、`quest`、`encounter_journal`、`minimap_button`、`chat_notify` 的运行时功能语义。
- 不改变现有 `ToolboxDB.modules.*` 的归属、键名语义或模块开关含义。
- 不在本轮引入新的控制台、仪表盘、收藏项、折叠导航体系或额外视觉皮肤。
- 不顺带扩展与本次信息架构重构无关的模块功能。
- 不新增新的小地图按钮功能入口；仅调整已有悬停菜单的设置方式与固定样式。

## 4. 已确认决策

- 选定设计方案为“纯叶子页”，不保留可点击总览页。
- 左侧树只保留一个根 `Toolbox` 与 6 个叶子页：`通用`、`界面`、`地图`、`任务`、`冒险手册`、`关于`。
- “通用”页承载语言设置、重载界面、`minimap_button` 与 `chat_notify`。
- `minimap_button` 的通用页设置收口为：
  - 主按钮仅保留圆形款式，不再提供方形选项。
  - 悬停菜单仅保留横向展开，不再提供纵向 / 横向切换。
  - 设置页中移除预览窗口、间距拖拽与功能池拖放。
  - 悬停菜单功能改为逐项勾选加入 / 移除；`flyoutSlotIds` 仅用于记忆勾选结果，不再承载手动排序。
- “界面”页承载 `mover` 与 `tooltip_anchor`；“地图”页承载 `navigation`；“任务”页承载 `quest` 与 Quest Inspector；“冒险手册”页承载 `encounter_journal`；“关于”页只保留静态说明。
- Quest Inspector 不再通过 `GetSettingsPages()` 暴露独立左侧子页，而是作为“任务”页里的低频工具区存在。
- `/toolbox`、ESC 菜单按钮和小地图按钮三种入口统一执行“优先回到上次停留叶子页，否则回退到 `通用`”的默认规则。
- 若需要记录宿主级设置状态，应优先落在 `ToolboxDB.global`，而不是创建新的 `modules.*` 存档。
- 旧存档中的 `buttonShape`、`flyoutExpand`、`flyoutLauncherGap`、`flyoutPad`、`flyoutGap` 视为已废弃键，需要在配置初始化阶段清理，避免残留值继续影响运行时。

## 5. 待确认项

- 无。用户已确认小地图按钮设置收口方案 A，并已明确回复“开动”。

## 6. 验收标准

1. 左侧设置树只保留一个根 `Toolbox` 与 6 个叶子页：`通用`、`界面`、`地图`、`任务`、`冒险手册`、`关于`。
2. 点击任一叶子页后，右侧直接进入最终设置内容，不再出现总览页、模块平铺首页或组内再导航。
3. “任务”页内直接包含 Quest Inspector，不再保留左侧独立 Quest Inspector 设置子页。
4. “通用”页至少收编语言设置、重载界面、小地图按钮和聊天加载提示；“关于”页仅保留静态说明，不承载可操作设置。
5. `/toolbox`、ESC 菜单按钮和小地图按钮三种入口行为一致，并遵循“上次停留页优先，否则回退到 `通用`”的规则。
6. 战斗内独立宿主模式仍可直接打开指定叶子页，且不会因移除总览页而退化。
7. 语言切换后，叶子页标题、说明和控件文案可完整重建；`debug` 与 `重置/重建` 在各叶子页中都位于低频区域。
8. “通用”页中的小地图按钮设置不再出现展开方式、按钮款式、预览窗口、功能池拖放或“全部加入”。
9. 小地图按钮实际运行时固定为圆形主按钮 + 横向悬停菜单；悬停菜单功能通过勾选项控制加入 / 移除，并在重载后保留。
10. 模块运行时行为、`moduleId` 与 `ToolboxDB.modules.*` 数据落点语义保持不变；其中 `flyoutSlotIds` 继续保留，但只表示已勾选的悬停菜单项集合。
11. 与设置结构相关的静态校验和全量测试通过，相关设计/功能文档已同步回写。

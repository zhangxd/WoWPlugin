# 设置宿主重构需求

- 文档类型：需求
- 状态：可执行
- 主题：settings-host-redesign
- 适用范围：`Toolbox/UI/SettingsHost.lua` 的设置控件承载方式、页内布局节奏、模块设置交互统一，以及现有 `RegisterSettings(box)` 协作模型的第二阶段重构
- 关联模块：`mover`、`tooltip_anchor`、`navigation`、`quest`、`encounter_journal`、`minimap_button`、`chat_notify`
- 关联文档：
  - `docs/designs/settings-host-redesign-design.md`
  - `docs/plans/settings-host-redesign-plan.md`
  - `docs/Toolbox-addon-design.md`
- 最后更新：2026-05-02

## 1. 背景

- `SettingsHost` 的纯叶子页结构已在上一阶段完成，当前左侧树、叶子页归属和默认打开规则已经收口。
- 但页内控件仍大量沿用旧式模板和手工交互：`InterfaceOptionsCheckButtonTemplate`、`UICheckButtonTemplate`、`UIDropDownMenuTemplate`、互斥勾选模拟单选等。
- 当前设置页的说明文字也偏多，常见模式是“标题 + 长说明 + 老控件散排”，与现行 Retail 设置页的阅读和操作节奏不一致。
- 2026-05-02 游戏内回看发现，当前第二阶段 helper 实现虽然统一了交互入口，但把 toggle / 单选 / 菜单 / 多选大量落成了 `UIPanelButtonTemplate` 风格按钮，导致系统设置页主观观感变成一排红底按钮，和“现代 Retail Settings”目标仍有明显偏差。
- 2026-05-02 用户进一步确认：`关于` 页不再暴露命令提示、仓库文档或 AI 协作流程，只保留面向玩家的公开插件信息。
- 本轮目标不是再改信息架构，而是在现有 6 个叶子页基础上，把页内布局和全部设置控件统一到更接近 WoW 现行 Settings 体验的口径，同时保持模块行为与存档语义不变。

## 2. 目标

- 保留现有 6 个叶子页和模块归属不变，只重构页内设置交互与布局节奏。
- 将普通布尔开关、单选、多选、下拉和操作按钮统一成同一套宿主级设置行风格。
- 移除冗余说明文字，让标题、辅助说明和控件保持稳定的一行式或近一行式节奏。
- 将 `关于` 页收口为公开对外说明，不再承载命令清单、内部文档路径或流程信息。
- 除真正的动作项外，页内设置不得再以红底按钮作为主视觉承载。
- 保持现有 `ToolboxDB` 键名、取值语义和模块行为不变。
- 保持现有设置刷新与即时生效链路不退化，并覆盖战斗内独立宿主模式。

## 3. 范围

### 3.1 In Scope

- 升级 `Toolbox/UI/SettingsHost.lua`，让宿主继续负责设置页注册的同时，统一页内设置行的构建方式。
- 保留 `RegisterSettings(box)` 契约，但将 `box` 升级为统一设置构建容器，并明确宿主 / 模块职责边界。
- 将各模块现有设置项统一到新交互风格，覆盖：
  - 普通布尔开关
  - 单选项
  - 多选项
  - 下拉 / 菜单选择
  - 操作按钮与简短说明
- 将以下现有旧式实现迁移为统一风格：
  - `SettingsHost` 的语言选择
  - `chat_notify` 的颜色下拉
  - `tooltip_anchor` 的模式单选
  - `mover` 的命中模式单选
  - `minimap_button` 的坐标锚点单选
  - `minimap_button` 的 flyout 多选列表
  - 各页已有 `enabled` / `debug` 等布尔开关
- 明确父子依赖、刷新方式、非法枚举值回退和战斗内独立宿主模式的兼容规则。
- 更新与设置结构和交互风格直接相关的文档与静态校验口径。

### 3.2 Out of Scope

- 不调整现有 6 个叶子页分组，不新增页面层级，不新增总览页或额外导航。
- 不新增模块、不新增玩家可见入口、不修改 `Toolbox/Toolbox.toc`。
- 不修改 `ToolboxDB` 键名、取值语义、模块归属和运行时功能行为。
- 不顺带重做插件视觉主题、仪表盘、控制台或其它超出设置页的界面。
- 不借本轮改造扩展与设置 UI 无关的模块功能。

## 4. 已确认决策

- 继续沿用当前纯叶子页结构，不改 `通用 / 界面 / 地图 / 任务 / 冒险手册 / 关于` 这 6 个叶子页。
- 现有 `ToolboxDB` 键名与取值语义完全兼容，只改 UI 层和设置绑定方式。
- 页内布局方向采用“原生列表式”：标题与简短说明在左，控件在右，整体节奏尽量贴近 Retail 新版设置页。
- 本轮统一范围包含普通布尔开关，不只处理单选 / 多选 / 下拉。
- 说明文字以“够用即可”为准，移除冗余长说明，只保留分节标题和必要辅助说明。
- 继续保留 `RegisterSettings(box)`，但 `box` 改为统一设置构建容器，由宿主负责布局与控件风格。
- 宿主负责：
  - 纵向游标与 `realHeight` 收口
  - 语言、reload 和页面级公共区块
  - 通用设置行、禁用态、间距和页底低频动作区
- 模块负责：
  - 自身设置项的数据读写与行为回调
  - 按宿主提供的 helper 声明模块专属设置项
  - 普通模块的 `RegisterSettings(box)` 不再承载公共 `enabled / debug / reset` 区
  - 仅在宿主 helper 覆盖不了的区域保留自定义内容块，并显式回报高度
- 接口边界：
  - 本轮继续沿用 `RegisterSettings(box)` 作为唯一模块设置入口，不新增独立的内容块注册接口
  - 复杂内容块仍通过 `RegisterSettings(box)` 挂到宿主提供的自定义块入口下
- 控件映射规则如下：
  - 布尔开关：统一为宿主级 toggle 行
  - 两项单选：优先收口为 segmented / pill 样式
  - 三项及以上单选：优先收口为 dropdown / menu button 样式
  - 多选项：统一为标准化多选列表
  - 下拉选择：不再使用 `UIDropDownMenu_*` 链路
  - 操作按钮：并入统一设置行节奏
- 用户 2026-05-02 明确确认本轮采用“方案 A：保留 6 个叶子页与 `RegisterSettings(box)` 契约，只重写 `SettingsHost` 行控件观感”的实施路径，不采用“只换皮现有按钮”或“深度混入新的 Blizzard Settings 原生控件注册链路”。
- 本轮代码实施在隔离 worktree 分支 `codex/settings-ui-modernize` 中完成，避免影响当前主工作区上未提交的导航改动；该约束属于执行边界，不改变插件运行时行为。
- 非法枚举值处理规则：
  - 单选 / 下拉读取到未知值时，UI 统一回退到该设置的默认项
  - 在页面构建阶段直接将该字段归一并写回默认值，避免出现双选中、无选中或占位状态
- 刷新规则：
  - 页面注册只在宿主初始化阶段执行一次，后续重建不重复注册类目或叶子页
  - 语言切换统一走 `RefreshAllPages()` 全量重建
  - 仅影响勾选态、文案或禁用态时，优先局部刷新当前行或当前组
  - 影响区块显隐、行数、`realHeight` 或自定义内容块高度时，统一走 `BuildPage()` 重建当前叶子页
- `关于` 页内容口径：
  - 仅保留版本、客户端和简短公开插件说明
  - 不显示 slash 命令、开发文档、仓库文件名或协作流程信息
- 战斗内独立宿主模式必须继续支持这些新控件；若某原生控件路径在独立宿主中不稳定，需要由宿主提供等价封装，不得退化为不可操作状态。
- 战斗内独立宿主模式的打开优先级如下：
  - 显式目标叶子页有效时，优先打开该页
  - 否则回到最近一次有效的 `settingsLastLeafPage`
  - 再否则回退到 `通用`
- 战斗内切换叶子页时，允许在已创建的独立宿主内部切页，并在切页成功后更新 `settingsLastLeafPage`；该路径不得重新注册类目，也不得调用系统 `Settings.OpenToCategory`。

## 6. 验收标准

1. 所有 6 个叶子页仍可正常打开，叶子页归属和默认打开规则保持不变。
2. 页内设置项统一为原生列表式节奏，不再保留互斥勾选模拟单选和 `UIDropDownMenu_*` 旧链路。
3. 普通布尔开关、单选、多选、下拉与操作按钮的样式和间距由宿主统一管理，不再由各模块各自散排。
4. 除真正的动作项外，主要设置控件不再以 `UIPanelButtonTemplate` 红底按钮作为默认观感；toggle、单选、菜单、多选均回到更接近 Retail Settings 的现代样式。
5. `RegisterSettings(box)` 的宿主 / 模块职责清晰：公共布局与高度收口由宿主负责，模块只负责设置声明和行为回调。
6. 现有设置都能继续修改、保存，并在 `/reload` 后保留原值。
7. 原本是单选的设置项不会出现双选中或无选中的非法状态；非法枚举值会回退并归一到默认项。
8. 相关模块改动后，保持原有即时生效或原有生效时机，不引入额外行为变化。
9. 页面滚动区域正常，无明显截断、重叠或间距错乱。
10. 战斗内独立宿主模式中，新控件仍可等价操作，不因控件升级而退化。
11. `python tests/run_all.py --ci` 继续通过。
12. `关于` 页只显示公开对外信息，不再出现命令列表、文档文件名或内部流程说明。

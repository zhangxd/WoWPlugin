# 冒险手册任务页签卡顿修复需求

## 1. 背景

- 模块：`encounter_journal`
- 现象：打开冒险手册“任务”页签时主线程明显卡顿；打开后拖动或滚动左侧滚动条时仍会卡顿。
- 影响范围：现有“任务”页签三视图中的树形浏览与详情联动，不新增入口、不新增模块、不修改 TOC。

## 2. 已确认结论

- 根因 1：`Toolbox.Questlines.GetQuestTabModel()` 在打开任务页签时同步做全量 strict 校验、任务线模型构建，以及大量任务运行时字段查询。
- 根因 2：左侧树当前默认展开地图、任务线、任务三级节点，首次进入会一次性构造大量行按钮，滚动时继续暴露同一批 UI 负担。
- 当前 live 数据量级：
  - 任务线地图数约 `239`
  - 任务线数约 `1563`
  - `questLineXQuest` 链接数约 `20718`

## 3. 用户已确认决策

- 确认时间：`2026-04-11`
- 用户回复：`开动`
- 选定主方案：**方案 A**

### 3.1 方案 A 内容

1. 任务模型拆分为“静态结构缓存”和“动态字段按需查询”两层。
2. 打开任务页签时不再为全部任务一次性补齐运行时字段。
3. 左侧树默认折叠地图和任务线，仅在用户展开时生成下一层节点。
4. 若存在已保存选中项，仅展开到恢复该选中项所必需的路径。
5. 保持现有模块边界，修改仍落在 `Toolbox/Core/API/QuestlineProgress.lua` 与 `Toolbox/Modules/EncounterJournal.lua`，不新建模块。

## 4. 边界

- 不做虚拟滚动列表重构。
- 不新增玩家可见入口。
- 不修改 `Toolbox/Toolbox.toc`。
- 不变更其他冒险手册增强功能（坐骑筛选、CD 文案、小地图悬停项等）的行为。

## 5. 验收标准

1. 打开任务页签时，不再因全量任务运行时查询与全树展开造成明显主线程卡顿。
2. 左侧滚动时，不再因一次性存在全部任务节点而持续明显卡顿。
3. 状态 / 类型 / 地图三视图仍可正常切换。
4. 地图、任务线、任务详情联动保持正确。
5. 已保存选择状态恢复后，仍能定位到上次选中的地图 / 任务线 / 任务。
6. 现有离线逻辑测试继续通过，并新增覆盖本次修复关键行为的回归测试。

## 6. 实施状态

- 状态：**已确认，可执行**
- 下一步：先补回归测试，再修改业务代码实现惰性展开与按需动态字段查询。

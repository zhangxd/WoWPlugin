# 冒险指南任务页签导航重构需求规格

- 文档类型：需求
- 状态：已完成
- 主题：encounter-journal-quest-navigation
- 适用范围：`encounter_journal` 任务页签的左侧导航树、主区折叠列表、任务类型入口、任务详情交互与相关存档迁移
- 关联模块：`encounter_journal`
- 关联文档：
  - `docs/designs/encounter-journal-design.md`
  - `docs/designs/encounter-journal-quest-navigation-design.md`
  - `docs/plans/encounter-journal-quest-navigation-plan.md`
  - `docs/Toolbox-addon-design.md`
- 最后更新：2026-04-12

## 1. 背景

- 当前任务页签上一轮已收口为顶部“资料片 / 分类”导航，但新一轮交互讨论中，用户进一步确认要把资料片固定为左侧一级导航，并把“地图任务线 / 任务类型”并入资料片展开树。
- 地图路径下，主区不再切成单独任务线页，而是要做“任务线单行列表 + 点击展开任务列表 + 再点折叠”的折叠式表现。
- 任务类型后续要支持地下城任务、团队副本任务等玩家可理解的大类，因此类型导航需要基于 `GetQuestTagInfo / Enum.QuestTagType` 的展示层归并，而不是直接暴露所有运行时碎类型。

## 2. 目标

- 将任务页签统一为左侧固定资料片一级导航的结构，并在资料片下提供两条浏览路径：`地图任务线` 与 `任务类型`。
- 让地图路径和类型路径职责分离：
  - 地图路径：`资料片 -> 地图任务线 -> 地图 -> 主区折叠任务线 -> 任务`
  - 类型路径：`资料片 -> 任务类型 -> 类型大类 -> 主区任务列表`
- 保持任务详情为“任务名列表 + 悬停 tooltip + 点击弹框”，并允许从任务详情回跳到对应资料片 / 地图 / 任务线。

## 3. 范围

### 3.1 In Scope

- 左侧一级导航固定为资料片列表，可上下滚动。
- 选中资料片后，在该资料片下展开两个固定入口：`地图任务线` 与 `任务类型`。
- 选中 `地图任务线` 后，左侧继续显示该资料片下的地图列表。
- 选中某个地图后，主区显示该地图下的任务线列表；每条任务线独占一行，点击展开其任务列表，再次点击折叠；同一时刻只展开一条任务线。
- 选中 `任务类型` 后，左侧继续显示该资料片下归并后的任务类型大类列表。
- 选中某个任务类型大类后，主区直接显示该类型下的任务列表。
- 任务列表仅显示任务名称，不默认展开详情。
- 任务名称悬停显示 tooltip，点击后显示任务详情弹框。
- 任务详情弹框在任务具备任务线归属时，提供“跳转到对应地图 / 任务线”的回跳入口。
- 主区顶部显示当前导航路径，样式参考冒险指南选择团队副本首领后的层级导航；路径中的上级节点可点击回退。
- 为任务线静态导出补充稳定的资料片归属字段，并在插件侧消费该字段。
- 为旧的顶部资料片 / 分类导航状态写新一轮迁移，切到左侧树模式后的状态模型。

### 3.2 Out of Scope

- 不新增 `RegisterModule` 模块。
- 不新增新的 slash 命令、设置入口或玩家可见功能入口。
- 不重构冒险指南副本列表“仅坐骑”、CD 叠加、详情页重置标签等非任务页签能力。
- 不在任务列表页默认展开所有任务详情。
- 不把任务线节点继续挂入左侧树的最终层级；任务线仍由主区承担展示和展开交互。
- 不在类型路径下增加“任务线 -> 任务”中间层。

## 4. 已确认决策

- 主归属模块保持为 `encounter_journal`，不拆新模块。
- 左侧一级菜单固定为资料片列表。
- 资料片下固定展开两个入口：`地图任务线` 与 `任务类型`；不再使用顶部“资料片 / 分类”双排按钮结构。
- 地图路径下，地图节点选中后，主区展示任务线单行列表；点击任务线展开任务列表，再点折叠；同一时刻只展开一条任务线。
- 任务类型路径下，类型节点选中后，主区直接展示任务列表，不增加任务线中间层。
- 任务类型展示优先使用 `GetQuestTagInfo / Enum.QuestTagType` 做玩家可读的大类分组，而不是直接暴露所有 `GetQuestType()` 碎类型。
- 任务列表页仅显示任务名；鼠标移动到任务上显示 tooltip；点击任务后弹框显示详细信息。
- 任务详情弹框中，若任务属于任务线，则提供“跳转到对应地图 / 任务线”入口。
- 主区顶部新增 breadcrumb 路径：
  - 地图路径下显示 `资料片 / 地图任务线 / 地图`，若当前展开了任务线，则追加 `任务线`
  - 类型路径下显示 `资料片 / 任务类型 / 类型大类`
- breadcrumb 中除最后一段外均可点击回退到对应层级。
- 一级“资料片”语义按“任务线实际所属资料片”处理，不按当前地图页签直接分桶。
- 资料片归属的数据来源选定为：在 `instance_questlines` 契约导出阶段，根据任务线主归属地图链路映射到 `Map.ExpansionID`，并将结果固化为任务线稳定字段，由插件侧直接消费，不在 UI 运行时临时猜测。
- 旧的任务树折叠状态与三视图选择状态不再作为主交互模型；新交互改为左侧导航路径 + 当前地图路径下的展开任务线选择。

## 5. 待确认项

- 无。主路径、类型视图职责、主区展开方式和回跳行为均已确认，可直接进入实现。

## 6. 验收标准

1. 在冒险指南“任务”页签中，左侧第一层始终是资料片列表，且可上下滚动。
2. 选中某个资料片后，左侧会出现 `地图任务线` 与 `任务类型` 两个固定入口。
3. 选中 `地图任务线` 后，左侧显示该资料片下的地图列表。
4. 选中某个地图后，主区显示该地图下的任务线列表；每条任务线独占一行。
5. 点击某条任务线后，仅展开该任务线的任务列表；再次点击同一任务线时折叠；切换到另一条任务线时，旧展开项自动收起。
6. 选中 `任务类型` 后，左侧显示归并后的任务类型大类；选中某个类型后，主区直接显示该类型下的任务列表。
7. 任务列表仅显示任务名称，不默认展开详细文本。
8. 鼠标悬停任务名称时，能看到任务 tooltip。
9. 点击任务名称后，出现可关闭的任务详情弹框，并显示完整任务详情。
10. 若任务存在任务线归属，任务详情弹框中能跳转到对应资料片 / 地图 / 任务线，并自动展开目标任务线。
11. 旧存档升级后不报错，且不会继续沿用“顶部资料片 / 分类导航”与更早的三视图状态。
12. 任务线资料片分组来自稳定导出字段，而不是插件运行时按地图名或当前页签猜测。
13. 主区顶部 breadcrumb 会随着当前层级变化，并且点击上级路径后能正确回退到对应状态。
